데이터베이스 기초 활용
[1] 데이터베이스는 다수의 인원, 시스템 또는 프로그램이 사용할 목적으로 통합하여 관리되는 (	1    ) 이다. 

[2] 테이터베이스 종류중 알맞는 설명끼리 묶인것을 고르세요 

1. 파일시스템 : 파일에 이름을 부여하고 저장이나 검색을 위하여 논리적으로 그것들을 어디에 위치시켜야 하는지 등을 정의한 뒤 관리하는 데이터 베이스 전 단계의 데이터 관리 방식이다. 
2. HDBMS : 데이터를 상하 종속적인 관계로 계층화하여 관리하는 데이터베이스 이다. 데
3. NDBMS : 데이터의 구조를 네트워크상의 망상 형태로 논리적으로 표현한 데이터 모델이다. 
4. RDBMS : 가장 보편화된 데이터베이스 관리시스템이다. 데이터를 저장하는 테이블의 일부를 다른 태이블과 상하 관계로 표시하지 않는다. 

[3] DBMS의 특징으로 틀린것은?
1. 데이터 무결성 : 부적절한 자료가 입력되어 동일한 내용에 대하여 서로 다른 데이터가 저장되는 것을 허용하지 않는 성질
2. 데이터 일관성 : 삽입, 삭제, 갱신, 생성 후에도 저장된 데이터가 변함없이 일정
3. 데이터 회복성 : 장애가 발생하였을 시 특정 상태로 복구되어야 하는 성질
4. 데이터 보안성 : 응답시간, 저장공간 활용 등이 최적화되어 사용자, 소프트웨어, 시스템 등의 요구 조건을 만족 시켜야 하는 성질

1. Entity

4.Variable

네트워크 기초 활용하기 

[1] OSI 7 Layer의 특징을 바르게 연결하세요

  물리 계층 *		* 송신, 수신 프로세스 간의 연결
  
  응용 게층 * 		* 실제 장비들을 연결하기 위한 연결 장치

  네트워크 계층 * 		* 사용자 친화 환경 제공 

  데이터 링크 계층 * 	* 송신, 수신 간의 논리적 연결

  표현 계층 * 		* 오류와 흐름을 제거하여 신뢰성 있는 데이터를 전송 

  전송 계층 * 		* 다소의 중개 시스템 중 올바른 경로를 선택하도록 지원

  세선 계층 * 		* 코드 문자 등을 번역하여 일관되게 전송하고 압축, 해제, 보안 기능도 담당

[2] 네트워크 장비의 역할중 틀린것은?

 1. 허브 : 다수의 오프라인, 온라인 접속 기기들을 로컬 네트워크에 연결하기 위한 장비
 2. 스위치 : MAC 주소 테이블을 이용하여 목적지 MAC 주소를 가진 장비 측 포트로만 프레임을 전송하는 역할을 한다. 
 3. 라우터 : LAN과 LAN을 연결하거나 LAN과 WAN을 연결하기 위한 인터넷 네트워킹 장비이다. 
 4. NIC : 내부 네트워크와 접속하여 가장 빠른 속도로 데이터를 주고받을 수 있게 컴퓨터 내에 설치되는 장치이다. 

[3] 네트워크 프로토콜의 특징중 아닌것은 
 1. 단편화
 2. 다중화
 3. 정기화
 4. 캡슐화

[4] TCP의 설명으로 옳은 것은? 
1. 연결되어 있어도 데이터를 송신할 수 있다. 단, 수신 측의 수신 여부는 확인하기 어렵다. 
2. Flow Control, Error Control을 하지 않아 신뢰성 있는 데이터 전송에는 부적합하다. 
3. 논리적인 1:1 가상 회선을 지원하여 해당 경로로만 데이터가 전달되도록 한다. 
4. 대표 서비스: SNMP, DNS, TFTP, NFS, NETBIOS, 인터넷 게임/방송/증권 등

[5] UDP의 설명으로 옳은 것은?
1 CRC 체크와 재전송 기능을 통해 신뢰성 있는 전송을 확보한다. 
2 Flow Control 기능을 수행하여 단계별 데이터 전송 상황을 체크한다. 
3 하나의 송신 정보를 다수의 인원이 수신해야 할 경우 UDP를 사용한다.
4 대표 서비스: FTP, Telnet, Http, SMTP, POP, IMAP 등

[6] 라우팅 프로토콜 RIP의 설명중 틀린것은?
1. 최초의 라우팅 프로토콜 
2.링크 상태 알고리즘 사용  
3. 30초 주기로 전체 라우팅 정보 갱신 
4. 변화 업데이트 시 많은 시간 소요 

[7] 패킷 스위칭의 설명 중 틀린것은?
1. 테이터의 단위를 보내는 방식
2. 회선 효율이 우수하다. 
3. 경로 접속 시간은 1초 내외로 매우 빠르다. 
4. 실시간 전송에 부적합하다. 

[8] 서킷 스위칭의 설명 중 틀린것은
1. 전송 경로를 설정한 뒤 데이터를 송수신하는 방법
2. 전송 제어 절차와 형식에 제약을 받지 않는다.
